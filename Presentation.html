Variables Exógenas

La serie de Halloween tiene estacionalidad marcada pero asimétrica: las visitas se aceleran conforme se acerca el 31-oct y caen abruptamente después, como el doble de un codo.

Por eso, en lugar de depender solo de valores pasados, construimos variables que le dan contexto temporal al modelo — no solo “qué pasó antes”, sino cuándo está ocurriendo y qué tan cerca está de Halloween.

Variables Exógenas — Tipos

Variables de calendario

Representan la cercanía al evento y patrones temporales: dias_para_halloween, es_halloween, semana_halloween, y variables cíclicas (sin/cos) para día de semana y mes.

Variables de memoria

Lags de 7 y 14 días capturan ciclicidad semanal. A largo plazo pasan a alimentarse de predicciones propias — la acumulación de error es menor aquí por la estabilidad de la serie.

```{python}
def days_to_halloween(date):
    hw = pd.Timestamp(date.year, 10, 31)
    return max((hw - date).days, 0)

# Halloween
df['dias_para_halloween'] = df.index.map(days_to_halloween)
df['es_octubre']          = (df.index.month == 10).astype(int)
df['es_halloween']        = ((df.index.month == 10) & (df.index.day == 31)).astype(int)
df['semana_halloween']    = ((df.index.month == 10) & (df.index.day >= 24)).astype(int)

# Fecha cíclica
df['es_fin_de_semana'] = (df.index.dayofweek >= 5).astype(int)
df['dia_semana_sin']   = np.sin(2 * np.pi * df.index.dayofweek / 7)
df['dia_semana_cos']   = np.cos(2 * np.pi * df.index.dayofweek / 7)
df['mes_sin']          = np.sin(2 * np.pi * df.index.month / 12)
df['mes_cos']          = np.cos(2 * np.pi * df.index.month / 12)

# Lags
df['views_lag7']  = df['views'].shift(7)
df['views_lag14'] = df['views'].shift(14)

df_model = df.dropna()
print(f"Shape final: {df_model.shape}")
```

Correlación de Pearson

Mide la fuerza de la relación lineal entre cada feature y las vistas diarias. Los valores van de −1 a 1: cercano a 1 = relación positiva fuerte, cercano a 0 = sin relación lineal.

Pearson solo detecta relaciones lineales y es sensible a valores extremos. Con los spikes de Halloween, algunas variables pueden parecer poco correlacionadas

Entrenamiento — Pesos Ponderados

El spike de Halloween representa solo unos pocos días dentro de años de datos. Sin ponderación, el modelo aprendería principalmente los días normales e ignoraría el evento más importante.

Durante el entrenamiento, cada observación contribuye al error total según su peso. Con sample_weights, el gradiente se escala por ese peso — un día con peso 15 produce un ajuste 15 veces mayor en los parámetros que un día normal.

Así se distribuyen los pesos en la serie:

  • Peso 1 — días normales (la mayoría del año)
  • Peso 3–4 — rampa: 26–30 de octubre
  • Peso 10–15 — spike: 31 de octubre
def make_sample_weights(dates, 
    spike_weight=15.0, 
    ramp_weight=3.0):
  w = np.ones(len(dates))
  for i, d in enumerate(dates):
    if d.month==10 and d.day==31:
      w[i] = spike_weight
    elif d.month==10 and d.day>=26:
      w[i] = ramp_weight
  return w

El efecto: el modelo es penalizado mucho más fuertemente cuando falla en el spike o en la rampa previa, forzándolo a aprender esos pocos días críticos aunque sean minoría en el dataset.

Entrenamiento — Parámetros

FFNN

  • lr = 1e-3
  • batch_size = 32
  • spike_weight = 15
  • ramp_weight = 3
  • epochs = 300, patience = 30

LSTM

  • lr = 3e-4
  • batch_size = 16
  • spike_weight = 10
  • ramp_weight = 4
  • epochs = 300, patience = 30

Note

¿Por qué Huber? Con MSE, los spikes de Halloween se castigarían de forma desproporcionada al elevar el error al cuadrado. Huber se comporta como MSE para errores pequeños y como MAE para errores grandes (delta=1.0), siendo más robusto ante outliers extremos.

Conclusiones

  • La transformación logarítmica sirvió mucho para que la serie también aprendiera el comportamiento general de la serie el resto del año
  • Las variables exógenas fueron fundamentales para ayudar a las redes neuronales a comprender el por qué del comportamiento de la serie
